# UI/UX QA Checklist

## Global

- [ ] AppShell consistent across all protected pages
- [ ] Sidebar grouped and not cluttered
- [ ] Topbar sticky and useful
- [ ] Breadcrumbs visible where needed
- [ ] Global search does not break layout
- [ ] Page max width applied
- [ ] White-first visual style followed
- [ ] No random colors
- [ ] No random shadows
- [ ] No inline styles

## Dashboard

- [ ] KPI strip clear
- [ ] No more than 6 KPI cards
- [ ] No more than 3 charts
- [ ] Today's actions visible
- [ ] Recent activity visible
- [ ] Staff augmentation status visible
- [ ] Works on mobile

## List Pages

- [ ] Header with title, description, primary action
- [ ] KPI strip where useful
- [ ] Toolbar with search and filters
- [ ] Max 8 visible columns
- [ ] Row actions in menu
- [ ] Loading state
- [ ] Empty state
- [ ] Error state
- [ ] Mobile behavior checked

## Detail Pages

- [ ] Sticky header
- [ ] Summary cards
- [ ] Tabs used
- [ ] No long unstructured page
- [ ] Activity timeline present where needed
- [ ] Documents tab present where needed
- [ ] Audit tab protected by RBAC

## Forms

- [ ] Create/edit uses slideover
- [ ] Required fields first
- [ ] Advanced fields collapsed
- [ ] Inline validation
- [ ] Sticky footer
- [ ] Cancel and Save visible
- [ ] Mobile full-screen behavior

## Admin

- [ ] Settings layout used
- [ ] Security pages readable
- [ ] Audit logs searchable
- [ ] AI settings clear
- [ ] Roles and permissions not overwhelming

## Responsiveness

- [ ] 375px mobile usable
- [ ] 768px tablet usable
- [ ] 1440px desktop polished
- [ ] Sidebar drawer works
- [ ] Tables do not break page
- [ ] Slideovers become full-screen on mobile

## Accessibility

- [ ] Buttons have labels
- [ ] Inputs have labels
- [ ] Tab order logical
- [ ] Focus states visible
- [ ] Color contrast acceptable
- [ ] Dialogs trap focus if implemented by UI library
